Source Code

These source code files are a reconstruction of Sundance, the Apollo 9 LM (Lunar Module) AGC (Apollo Guidance Computer) flight software, created from binary dumps of original core rope program modules, as follows:
  • B1 — Revision 292
  • B2 — Revision 302
  • B3 — Revision 302
  • B4 — Revision 302
  • B5 — Revision 292
  • B6 — Revision 306
This means that source code does not represent a specific historical revision of Sundance, but rather a mixture of the three released versions. Nonetheless, it is capable of flying the whole Apollo 9 mission. This source code matches the binary dumps, as closely as feasible, but 100% match is not possible, due mostly to changes in memory addresses at which code and variables were allocated in different revisions of Sundance. Refer to this list of explanations for each binary mismatch.

Since only binary dumps (rather than listings) of Sundance are available as source material, all comments and labels are approximate. They have been taken from other AGC programs where possible, or, in some places, written from scratch to match what we believe would have been in the original listing.

050018,000020:                                                                                                  #  MOD BY - GAUNTT

050020,000022:                                                                                                  #  FUNCTIONAL DESCRIPTION
050021,000023:                                                                                                  #        PROGRAM HAS TWO MAIN PARTS. THE FIRST IS SELF-CHECK WHICH RUNS AS A ZERO PRIORITY JOB WITH NO CORE SET, AS
050022,000024:                                                                                                  #  PART OF THE BACK-UP IDLE LOOP. THE SECOND IS SHOW-BANKSUM WHICH RUNS AS A REGULAR EXECUTIVE JOB WITH ITS OWN
050023,000025:                                                                                                  #  STARTING VERB.
050024,000026:                                                                                                  #        THE PURPOSE OF SELF-CHECK IS TO CHECK OUT VARIOUS PARTS OF THE COMPUTER AS OUTLINED BELOW IN THE OPTIONS.
050025,000027:                                                                                                  #        THE PURPOSE OF SHOW-BANKSUM IS TO DISPLAY THE SUM OF EACH BANK, ONE AT A TIME.
050026,000028:                                                                                                  #        IN ALL THERE ARE 7 POSSIBLE OPTIONS IN THIS BLOCK II VERSION OF SELF-CHECK. MORE DETAIL DESCRIPTION MAY BE
050027,000029:                                                                                                  #  FOUND IN E-2065 BLOCK II AGC SELF-CHECK AND SHOW BANKSUM BY EDWIN D. SMALLY DECEMBER 1966, AND ADDENDA 2 AND 3.
050028,000030: 
050029,000031:                                                                                                  #        THE DIFFERENT OPTIONS ARE CONTROLLED BY PUTTING DIFFERENT NUMBERS IN THE SMODE REGISTER (NOUN 27). BELOW IS
050030,000032:                                                                                                  #   A DESCRIPTION OF WHAT PARTS OF THE COMPUTER THAT ARE CHECKED BY THE OPTIONS, AND THE CORRESPONDING NUMBER, IN
050031,000033:                                                                                                  #  OCTAL, TO LOAD INTO SMODE.
050032,000034:                                                                                                  #  +-4   ERASABLE MEMORY
050033,000035:                                                                                                  #  +-5   FIXED MEMORY
050034,000036:                                                                                                  #  +-1,2,3,6,7,10   EVERYTHING IN OPTIONS 4 AND 5.
050035,000037:                                                                                                  #  -0    SAME AS +-10 UNTIL AN ERROR IS DETECTED.
050036,000038:                                                                                                  #  +0    NO CHECK, PUTS COMPUTER INTO THE BACKUP IDLE LOOP.
050037,000039: 
050038,000040:                                                                                                  #  WARNINGS
050039,000041:                                                                                                  #        USE OF E MEMORY RESERVED FOR SELF-CHECK (EVEN IN IDLE LOOP) AS TEMP STORAGE BY OTHER PROGRAMS IS DANGEROUS.
050040,000042:                                                                                                  #        SMODE SET GREATER THAN OCT 10 PUTS COMPUTER INTO BACKUP IDLE LOOP.
050041,000043: 
050042,000044:                                                                                                  #  CALLING SEQUENCE
050043,000045:                                                                                                  #        TO CALL SELF-CHECK KEY IN
050044,000046:                                                                                                  #                V 21 N 27 E     OPTION NUMBER E
050045,000047:                                                                                                  #        TO CALL SHOW-BANKSUM KEY IN
050046,000048:                                                                                                  #                V 91 E          DISPLAYS FIRST BANK
050047,000049:                                                                                                  #                V 33 E          PROCEED, DISPLAYS NEXT BANK
050048,000050: 
050049,000051:                                                                                                  #  EXIT MODES, NORMAL AND ALARM
050050,000052:                                                                                                  #        SELF-CHECK NORMALLY CONTINUES INDEFINITELY UNLESS THERE IS AN ERROR DETECTED. IF SO + OPTION NUMBERS PUT
050051,000053:                                                                                                  #  COMPUTER INTO BACKUP IDLE LOOP, - OPTION NUMBERS RESTART THE OPTION.
050052,000054:                                                                                                  #        THE -0 OPTION PROCEEDS FROM THE LINE FOLLOWING THE LINE WHERE THE ERROR WAS DETECTED.
050053,000055:                                                                                                  #        SHOW-BANKSUM PROCEEDS UNTIL A TERMINATE IS KEYED IN (V 34 E).  THE COMPUTER IS PUT INTO THE BACKUP IDLE LOOP
050054,000056: 
050055,000057:                                                                                                  #  OUTPUT
050056,000058:                                                                                                  #        SELF-CHECK UPON DETECTING AN ERROR LOADS THE SELF-CHECK ALARM CONSTANT (01102) INTO THE FAILREG SET AND
050057,000059:                                                                                                  #  TURNS ON THE ALARM LIGHT. THE OPERATOR MAY THEN DISPLAY THE THREE FAILREGS BY KEYING IN V 05 N 09 E. FOR FURTHER
050058,000060:                                                                                                  #  INFORMATION HE MAY KEY IN V 05 N 08 E, THE DSKY DISPLAY IN R1 WILL BE ADDRESS+1 OF WHERE THE ERROR WAS DETECTED,
050059,000061:                                                                                                  #  IN R2 THE BBCON OF SELF-CHECK, AND IN R3 THE TOTAL NUMBER OF ERRORS DETECTED BY SELF-CHECK SINCE THE LAST MAN
050060,000062:                                                                                                  #  INITIATED FRESH START (SLAP1).
050061,000063: 
050062,000064:                                                                                                  #        SHOW-BANKSUM STARTING WITH BANK 0 DISPLAYS IN R1 THE BANK SUM (A +-NUMBER EQUAL TO THE BANK NUMBER), IN R2
050063,000065:                                                                                                  #  THE BANK NUMBER, AND IN R3 THE BUGGER WORD.
050064,000066: 
050065,000067:                                                                                                  #  ERASABLE INITIALIZATION REQUIRED
050066,000068:                                                                                                  #        ACCOMPLISHED BY FRESH START
050067,000069:                                                                                                  #                SMODE SET TO +0
050068,000070: 
050069,000071:                                                                                                  #  DEBRIS
050070,000072:                                                                                                  #        ALL EXITS FROM THE CHECK OF ERASABLE (ERASCHK) RESTORE ORIGINAL CONTENTS TO REGISTERS UNDER CHECK.
050071,000073:                                                                                                  #  EXCEPTION IS A RESTART. RESTART THAT OCCURS DURING ERASCHK RESTORES ERASABLE, UNLESS THERE IS EVIDENCE TO DOUBT
050072,000074:                                                                                                  #  E MEMORY, IN WHICH CASE PROGRAM THEN DOES A FRESH START (DOFSTART).
